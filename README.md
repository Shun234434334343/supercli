## 🛠️ Using supercli

supercli uses a simple configuration system to control how it works. You do not need to learn programming to set it up.

### Basic commands

- To see available commands, type:

  ```
  supercli help
  ```

- To run a task or feature, use:

  ```
  supercli run [task-name]
  ```

For example, to run a built-in skill, type the skill’s name after `run`.

### Configuring supercli

supercli reads settings from a file named `config.yml`. This file tells it how to behave and what tasks to run.

- The config file is in plain text, easy to edit with Notepad.

- You can find a sample `config.yml` in the installation folder or download one from the release page.

- Edit the file to change commands, enable or disable skills, or adjust settings.

Example snippet from `config.yml`:

```yaml
skills:
  greet:
    enabled: true
    message: "Hello from supercli!"
```

Change the message or add new skills to fit your needs.
